# 3. 特征缩放
scaler = StandardScaler()

# 6. 交叉验证
cv_scores = cross_val_score(logreg, X_train, y_train, cv=5, scoring='accuracy')

# 7. 超参数调优
pipeline = Pipeline([
    ('scaler', StandardScaler()),
    ('lr', LogisticRegression(max_iter=10000, random_state=42))
])

param_grid = {
    'lr__C': np.logspace(-4, 4, 20),
    'lr__penalty': ['l1', 'l2'],
    'lr__solver': ['liblinear']
}

grid_search = GridSearchCV(pipeline, param_grid, cv=5, scoring='accuracy', n_jobs=-1)
grid_search.fit(X_train, y_train)
results = pd.DataFrame(grid_search.cv_results_)

# 绘制超参数调优过程图并添加姓名
plt.figure(figsize=(12, 6))
for penalty in ['l1', 'l2']:
    subset = results[results['param_lr__penalty'] == penalty]
    plt.plot(subset['param_lr__C'], subset['mean_test_score'], 
             label=f'{penalty}正则化')

plt.xscale('log')
plt.xlabel('正则化强度 (C)')
plt.ylabel('交叉验证准确率')
plt.title(f'逻辑回归超参数调优 - {student_name}')  # 使用f-string添加姓名
plt.legend()
plt.grid(True)
plt.savefig('logistic_hyperparameter_tuning.png', dpi=300, bbox_inches='tight')
plt.show()

性能评估逻辑：ROC 曲线通过展示不同阈值下 FPR 与 TPR 的权衡关系，全面评估模型的分类能力；AUC 值则用单一指标量化模型对正负样本的排序能力。 适用场景：尤其适合类别不平衡数据（如本实验中良性样本多于恶性样本），避免仅用准确率导致的评估偏差。 结果解读：若 ROC 曲线接近左上角且 AUC≈1，说明模型性能优异；若接近对角线，则模型效果接近随机猜测。

评估价值：混淆矩阵通过分解四种分类结果，揭示模型在不同类别上的表现。在医疗场景中，FP（误诊）和 FN（漏诊）的代价不同，需特别关注。 业务洞察：

对角线值（TN 和 TP）越高，模型整体准确性越好。 非对角线值（FP 和 FN）反映错误类型： FP 高：模型倾向于过度诊断（假警报多）。 FN 高：模型倾向于漏诊（更危险，尤其在医疗场景）。

后续优化：若 FN 过高，可调整分类阈值（如降低阈值使模型更敏感），或采用代价敏感学习（对 FN 赋予更高权重）。

特征筛选依据：逻辑回归系数的绝对值反映了特征对分类决策的影响程度，绝对值越大，特征越重要。 业务价值：

特征解释：例如，在医疗诊断中，texture_worst（最差纹理）和radius_se（半径标准误差）等特征的高重要性，可能提示医生在临床判断时应重点关注这些指标。 特征工程：可根据重要性删除低贡献特征，简化模型或减少噪声。

局限性：

系数重要性仅适用于线性关系，若特征与类别存在非线性关系，需结合非线性模型（如随机森林）进行分析。 特征间的多重共线性可能导致系数不稳定，需预先处理相关性高的特征。
